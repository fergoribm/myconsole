My Console is an experimentation looking at a cross-region console, offering fast lookup and navigation of resources across regions, together with resource tagging.

Start by setting a token

The token is used to interact with Cloud Foundry APIs. Without a token, you can't retrieve your Cloud Foundry orgs, spaces, apps and services. Set your token here. Then click Refresh All.

Pick a visualization

Use the toolbar on the left to switch between different visualizations of the resources.

Then search for resources

Type any text in the Search bar to find resources.

The search supports a few keywords. It always performs a full inclusive search, i.e the resource must match all criteria.

  • region to focus on resources in a specific region. Possible values are ng, eu-gb, au-syd.
  • org to focus on resources in a specific organization
  • space to focus on resources in a specific space
  • service to select service instances of a specific service type
  • tag to select resources with the given tag
  • diego to select apps based on whether they have been deployed on the Diego architecture
  • url to select apps matching a given route
  • status to select apps with a given status
  • updated_at to select resources after a given date, updated_at:2016-12-31

A few examples of what you can search on:

  • status:down org:John to view all apps currently down in the organizations having their name containing John across all regions
  • service:cloudant to view all Cloudant service instances
  • todo to find all resources having todo in their name

Behind the scenes

My Console retrieves all your organizations, spaces, apps and services across regions and stores this information in the local storage of your web browser. Thus the information is not always up-to-date but good enough for a global snapshot of your resources and to navigate the regions, orgs and spaces.

Data is stored in a PouchDB, in the browser local storage. Client is built with Angular + Material.
